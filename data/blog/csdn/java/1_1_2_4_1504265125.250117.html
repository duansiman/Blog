默认Eclipse IDE: Eclipse IDE for Java EE Developers


一、首先讲解正确的方式，由正确的方式推导出解决方案

1、servers runtime:tomcat6.0  对应 Dynamic web module version 2.5


2、 项目工程的Build Path:

1）Source-工程编译后的路径( 1、WebContent/WEB-INF/classes  2、WebRoot/WEB-INF/classes )，一般由Myeclipse创建的工程是WebRoot，Eclipse创建的工程是WebContent，要注意有些配置文件的路径问题

2）Libraries:加入servers runtime,使用统一的Jre

3）修改工程编码，统一为UTF-8/GBK/GB2312



二、常见问题

迁出SVN下的JavaWeb工程，可选择为Dynamic web Project，根据项目需要的servers runtime，选择Dynamic web module version（可解决MyEclipse项目导入Eclipse不能被servers识别的问题，对应第3条）

1、servlet报红：找不到对应的servers runtime library（jsp-api.jar，servlet-api.jar）

2、引用Jar文件报错：使用项目规定的JDK版本，更改编译时的Jre版本；WEB-INF/lib下的Jar包，Add to Build Path；修改工程编码

3、servers不能加载项目：相应的工程版本对应相应的运行服务器

4、servers运行不成功，检查application*.xml,struts.xml,log4j.properties,jdbc.propertioes,oscache.xml.system.xml,memcached.xml,hibernate.cfg.xml,*.ftl,*.xml,*.properties,*.sql,查看数据库连接，缓存服务器，spring配置文件等等；排查数据库启动内存分配异常，排查程序启动时内存分配异常

5、Debug模式启动，由Web.xml启动顺序，断点跟踪

6、启动无敌模式，百度、Google。
