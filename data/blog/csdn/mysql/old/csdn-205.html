\n首先，你必须要有操作系统的root权限了。要是连系统的root权限都没有的话，先考虑root系统再走下面的步骤。
\n类似于安全模式登录系统，有人建议说是pkill mysql，但是我不建议哈。因为当你执行了这个命令后，会导致这样的状况：
\n/etc/init.d/mysqld status
\nmysqld dead but subsys locked
\n这样即使你是在安全模式下启动mysql都未必会有用的，所以一般是这样/etc/init.d/mysqld stop，如果你不幸先用了pkill，那么就start一下再stop咯。
\n# mysqld_safe --skip-grant-tables &
\n&，表示在后台运行，不再后台运行的话，就再打开一个终端咯。
\n# mysql
\nmysql> use mysql;
\nmysql> UPDATE user SET password=password("test123") WHERE user='root';   
\nmysql> flush privileges;
\nmysql> exit;
