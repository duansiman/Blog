       一存储过程简介

\n

       在学习存储过程之前我们先来看看MySQL数据库中的命令执行流程：

\n


\n

\n

       (1)存储过程

\n

       存储过程是SQL语句和控制语句的预编译集合，以一个名称存储并作为一个单元处理。

\n

       (2)存储过程的优点

\n

       1)增强SQL语句的功能和灵活性。

\n

       2)实现较快的执行速度。

\n

       3)减少网络流量。

\n

       (3)存储过程语法结构解析

\n       MySQL数据库创建存储过程的语法结构为：
\n \n

       CREATE [DEFINER = { user | CURRENT_USER} ] PROCEDURE sp_name ([proc_parameter[,...]]) 

\n

[characteristic ...] routine_body;

\n

       proc_parameter代表的含义：

\n       [ IN | OUT | INOUT ] param_name type
\n \n

       参数含义：

\n

       1)IN，表示该参数的值必须在调用存储过程时指定。

\n

       2)OUT，表示该参数的值可以被存储过程改变，并且可以返回。

\n

       3)INOUT，表示该参数的值在调用时指定，并且可以被改变和返回。

\n

       (4)存储过程特性

\n

       COMMENT 'string'

\n       | { CONTAINS SQL | NO SQL | READS SQL DATA | MODIFIES SQL DATA} 
\n       | SQL SECURITY { DEFINER | INVOKER }
\n \n

       特性释义：

\n

       1)COMMENT：注释。

\n

       2)CONTAINS SQL：包含SQL语句，但不包含读或写数据的语句。

\n

       3)NO SQL：不包含SQL语句。

\n

       4)READS SQL DATA ：包含读数据的语句。

\n

       5)MODIFIES SQL DATA：包含写数据的语句。

\n       6)SQL SECURITY { DEFINER | INVOKER }：指明谁有权限来执行。
\n \n

       (5)过程体

\n

       1)过程体由合法的SQL语句构成。

\n

       2)过程体可以是任意SQL语句。

\n       3)过程体如果为复合结构则使用BEGIN...AND语句。
\n \n

       4)复合结构也可以包含声明、循环和控制结构。

\n

       (6)调用存储过程

\n

       MySQL数据库调用存储过程的语法为：

\n

       1)调用有参数的存储过程

\n

       CALL sp_name([parameter[,...]]);

\n

       2)调用无参数的存储过程

\n

       CALL sp_name[()];

\n

       二创建存储过程

\n

       (1)创建不带参数的存储过程

\n

       创建查询MySQL数据库版本号的存储过程

\n       CREATE PROCEDURE sp1() SELECT VERSION();
\n \n

       调用存储过程

\n

       CALL sp1;

\n

       CALL sp1();

\n

       三存储过程与自定义函数的区别 

\n

       存储过程和自定义函数的区别：

\n

       1)存储过程实现的功能要复杂一些；而函数的针对性更强。

\n

       2)存储过程可以返回多个值；函数只能有一个返回值。

\n       3)存储过程一般独立的来执行；而函数可以作为其他SQL语句的组成部分来出现。
\n \n

       使用存储过程注意事项

\n

       1)创建存储过程或者自定义函数时需要通过DELIMITER语句修改定界符。

\n

       2)如果函数体或过程体有多个语句，需要包含在BEIGIN...AND语句块中。

\n

       3)存储过程通过CALL关键字来调用。
