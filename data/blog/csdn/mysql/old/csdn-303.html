\nRedis 数据类型

\n

\nRedis支持五种数据类型：string（字符串），hash（哈希），list（列表），set（集合）及zset(sorted set：有序集

\n

\n合)。

\n
\n

\nString（字符串）

\n

\nstring是redis最基本的类型，你可以理解成与Memcached一模一样的类型，一个key对应一个value。

\n

\nstring类型是二进制安全的。意思是redis的string可以包含任何数据。比如jpg图片或者序列化的对象 。

\n

\nstring类型是Redis最基本的数据类型，一个键最大能存储512MB。

\n

\n实例

\n
127.0.0.1:6379> SET name "runoob"\nOK\n127.0.0.1:6379> GET name\n"runoob"
\n

\n在以上实例中我们使用了 Redis 的 SET 和 GET 命令。键为 name，对应的值为 runoob。

\n

\n注意：一个键最大能存储512MB。

\n
\n

\nHash（哈希）

\n

\nRedis hash 是一个键值对集合。

\n

\nRedis hash是一个string类型的field和value的映射表，hash特别适合用于存储对象。

\n

\n实例

\n
127.0.0.1:6379> HMSET user:1 username runoob password runoob points 200\nOK\n127.0.0.1:6379> HGETALL user:1\n1) "username"\n2) "runoob"\n3) "password"\n4) "runoob"\n5) "points"\n6) "200"
\n

\n以上实例中 hash 数据类型存储了包含用户脚本信息的用户对象。 实例中我们使用了 Redis HMSET, HGETALL 命令，user:1为键值。

\n每个 hash 可以存储 232 -1 \n 键值对（40多亿）。 \n

\n

\n
\n

\nList（列表）

\n

\nRedis 列表是简单的字符串列表，按照插入顺序排序。你可以添加一个元素到列表的头部（左边）或者尾部（右边）。

\n

\n实例

\n
127.0.0.1:6379> lpush runoob redis\n(integer) 1\n127.0.0.1:6379> lpush runoob mongodb\n(integer) 2\n127.0.0.1:6379> lpush runoob rabitmq\n(integer) 3\n127.0.0.1:6379> lrange runoob 0 10\n1) "rabitmq"\n2) "mongodb"\n3) "redis"\n127.0.0.1:6379>
\n

\n列表最多可存储 232 - 1 元素 (4294967295, 每个列表可存储40多亿)。

\n
\n

\nSet（集合）

\n

\nRedis的Set是string类型的无序集合。

\n

\n集合是通过哈希表实现的，所以添加，删除，查找的复杂度都是O(1)。

\n

\nsadd 命令

\n

\n添加一个string元素到,key对应的set集合中，成功返回1,如果元素已经在集合中返回0,key对应的set不存在返回错误。

\n
sadd key member
\n

\n实例

\n
127.0.0.1:6379> sadd runoob redis\n(integer) 1\n127.0.0.1:6379> sadd runoob mongodb\n(integer) 1\n127.0.0.1:6379> sadd runoob rabitmq\n(integer) 1\n127.0.0.1:6379> sadd runoob rabitmq\n(integer) 0\n127.0.0.1:6379> smembers runoob\n\n1) "rabitmq"\n2) "mongodb"\n3) "redis"
\n

\n注意：以上实例中 rabitmq 添加了两次，但根据集合内元素的唯一性，第二次插入的元素将被忽略。

\n

\n集合中最大的成员数为 232 - 1(4294967295, 每个集合可存储40多亿个成员)。

\n
\n

\nzset(sorted set：有序集合)

\nRedis zset 和 set 一样也是string类型元素的集合,且不允许重复的成员。 \n

\n

\n

\n不同的是每个元素都会关联一个double类型的分数。redis正是通过分数来为集合中的成员进行从小到大的排序。

\n

\nzset的成员是唯一的,但分数(score)却可以重复。

\n

\nzadd 命令

\n

\n添加元素到集合，元素在集合中存在则更新对应score

\n
zadd key score member
\n

\n实例

\n
127.0.0.1:6379> zadd runoob 0 redis\n(integer) 1\n127.0.0.1:6379> zadd runoob 0 mongodb\n(integer) 1\n127.0.0.1:6379> zadd runoob 0 rabitmq\n(integer) 1\n127.0.0.1:6379> zadd runoob 0 rabitmq\n(integer) 0\n127.0.0.1:6379> ZRANGEBYSCORE runoob 0 1000\n\n1) "redis"\n2) "mongodb"\n3) "rabitmq"
