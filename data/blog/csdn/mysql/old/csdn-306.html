\nsortedset数据结构
\n
\nsortedset和set类型极为类似，它们都是字符串的集合，都不允许重复的成员出现在一个set集合中。它们之间的主要差别是sortedset中的每一个成员都会有一个分数（score）与之关联，redis正是通过分数来为集合指那个的成员进行从大到小的排序。然而需要额外指出的是尽管sortedset中的成员必须是唯一的，但是分数却是可以重复的。
\n
\n在sortedset中添加、删除或更新一个成员都是非常快速的操作。其时间复杂度为集合中成员数量的对数。由于sortedset中的成员在集合中的位置是有序的。因此，即便是访问位于集合中部的成员也仍然是非常高效的。
\n

\n常用命令
\n
\n添加元素
\n
\nzadd key score1 member1 score2 member2 ...
\n
\n将所有成员以及该成员的分数存放到sortedset中。如果该元素已经存在，则会用心的分数替换原有的分数，返回值时新加入到集合中的元素个数，不包含之前已经存在的元素。
\n
\n
\n
\n
\n
\n
\n获取元素
\n
\nzscore key member
\n
\n获取指定成员的分数
\n
\n
\n
\n
\n
\n
\nzcard key
\n
\n获取集合中的成员数量
\n
\n
\n
\n
\n
\n
\n删除元素
\n
\nzrem key member1 member2 ...
\n
\n移除集合中指定的成员，可以指定多个成员
\n
\n
\n
\n
\n
\n范围查询
\n
\nzrange key start end [withscores]
\n
\n获取集合中下标为stert到end的成员，[withscore]参数表名返回的成员包含其分数。
\n
\n
\n
\n
\n
\n
\nzrevrange key start stop [withscores]
\n
\n按照元素分数从大到小的顺序返回下标从start到stop之间的所有元素，包含两端的元素，[withscore]参数表名返回的成员包含其分数。
\n
\n
\n
\n
\n
\n
\nzremrangebyrank key start stop
\n
\n按照排名范围删除元素
\n
\n
\n
\n
\n
\n
\nzremrangebyscore key start stop
\n
\n按照分数范围删除元素
\n
\n
\n
\n
\n
\n
\nzrangebyscore key min max [withscores] [limit offset count]
\n
\n返回分数在[min,max]的成员并按照分数从低到高排序。[withscores]表示显示分数，[limit offset count]中的ofsfet表示从下标offset的元素开始并返回count个成员。
\n
\n
\n
\n
\n
\n
\nzincrby key increment member
\n
\n设置指定成员的增加的分数，返回值是更改后的分数
\n
\n
\n
\n
\n
\n
\nzcount key min max
\n
\n获取分数在[min,max]之间的成员
\n
\n
\n
\n
\n
\n
\nzrank key member
\n
\n从小到大返回成员在集合中的排名
\n
\nzrevrank key member
\n
\n从大到小返回成员在集合中的排名
\n

\nkeys通用操作

\nkeys pattern
\n

\nkeys *
\n
\nkeys my*
\n
\nkeys ??list
\n
\nkeys my???
\n
\n
\n
\n
\n
\n
\ndel key1 key2 ...
\n
\n删除指定的key
\n
\n
\n
\n
\n
\n
\nexists key
\n
\n判断该key是否存在，1代表存在，0代表不存在
\n
\n
\n
\n
\n
\n
\nrename key newkey
\n

\n
\n
\n
\n
\n

\n
