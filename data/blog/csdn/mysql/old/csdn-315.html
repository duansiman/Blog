Linux命令行成功登录mysql，执行命令报 Ignoring query to other database。

\n

[mysql@bjdev01 ~]$ mysql -Uroot -p
\nEnter password:
\nmysql> select now();
\nIgnoring query to other database
\nmysql> exit
\nBye

\n

 

\n

退出后重新登录，正常。
\n[mysql@bjdev01 ~]$ mysql -uroot -p
\nEnter password:

\n

mysql> select now();
\n+---------------------+
\n| now()               |
\n+---------------------+
\n| 2014-01-27 10:29:47 |
\n+---------------------+
\n1 row in set (0.00 sec)

\n

对比两次登录命令：
\n第一次登录： mysql -Uroot   -p
\n第二次登录： mysql -uroot    -p

\n

 

\n

发现使用的参数一个是大写U,一个是小写u，而这两个参数的含义是不一样的。

\n

查看了mysql的帮助，发现小写u和大写U代表不同的含义
\n[mysql@bjdev01 ~]$ mysql --help| egrep 'user|safe-updates'
\n  -u, --user=name     User for login if not current user.
\n  -U, --safe-updates  Only allow UPDATE and DELETE that uses keys.
\n  -U, --i-am-a-dummy  Synonym for option --safe-updates, -U.
\n  --select-limit=#    Automatic limit for SELECT when using --safe-updates.
\n                      --safe-updates.
\nuser                              (No default value)
\nsafe-updates                      FALSE

\n

如果加-U 登录mysql，则对数据的更新和删除一定要加条件。
