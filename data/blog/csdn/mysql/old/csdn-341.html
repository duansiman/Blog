2、设置自增列

\nMYSQL的自增列一定要是有索引的列，设置种子值要在表的后面设置
\n
 --mysql\n -- 设置自增ID从N开始\n CREATE TABLE emp (\n ID INT PRIMARY KEY AUTO_INCREMENT\n ) AUTO_INCREMENT = 100 ; --（设置自增ID从100开始）
设置自增列的步长，可以分为全局级别和会话级别,如果是会话级别，那么当用户新建一个会话的时候，那么步长又回到了全局级别mysql不能设置为表级别的步长！！
\nmysql服务器维护着2种mysql的系统参数（系统变量）：全局变量（global variables）和会话变量（session variables）。
\n它们的含义与区别如其各占的名称所示，session variables是在session级别的，对其的变更只会影响到本session；global variables是系统级别的，
\n对其的变更会影响所有新session（变更时已经存在session不受影响）至下次mysql server重启动。
\n注意它的变更影响不能跨重启，要想再mysql server重启时也使用新的值，那么就只有通过在命令行指定变量选项或者更改选项文件来指定，
\n而通过SET变更是达不到跨重启的。
\n每一个系统变量都有一个默认值，这个默认值是在编译mysql系统的时候确定的。
\n对系统变量的指定，一般可以在server启动的时候在命令行指定选项或者通过选项文件来指定
\n当然，大部分的系统变量，可以在系统的运行时，通过set命令指定其值。
\n查看系统当前默认的自增列种子值和步长值
\n
SHOW GLOBAL VARIABLES LIKE 'auto_incre%'; -- 全局变量
问：如果有一张表，里面有个字段为id的自增主键，当已经向表里面插入了10条数据之后，删除了id为8，9，10的数据，再把mysql重启，
\n之后再插入一条数据，那么这条数据的id值应该是多少，是8，还是11？
\n答：如果表的类型为MyISAM，那么是11。如果表的类型为InnoDB，则id为8。
\n这是因为两种类型的存储引擎所存储的最大ID记录的方式不同，MyISAM表将最大的ID记录到了数据文件里，重启mysql自增主键的最大ID值也不会丢失；
\n而InnoDB则是把最大的ID值记录到了内存中，所以重启mysql或者对表进行了OPTIMIZE操作后，最大ID值将会丢失。
\n顺便说一下MYSQL获取当前表的自增值的四种方法
\n
(1) SELECT MAX(id) FROM person 针对特定表\n(2) SELECT LAST_INSERT_ID() 函数 针对任何表\n(3) SELECT @@identity 针对任何表
@@identity 是表示的是最近一次向具有identity属性(即自增列)的表插入数据时对应的自增列的值，是系统定义的全局变量。
\n一般系统定义的全局变量都是以@@开头，用户自定义变量以@开头。
\n使用@@identity的前提是在进行insert操作后，执行select @@identity的时候连接没有关闭，否则得到的将是NULL值。
\n
(4) SHOW TABLE STATUS LIKE 'person'
如果针对特定表，建议使用这一种方法
\n得出的结果里边对应表名记录中有个Auto_increment字段，里边有下一个自增ID的数值就是当前该表的最大自增ID.

4、修改表名

\n
ALTER TABLE emp RENAME emp2
\n

5、修改字段的数据类型

\n    将id字段的int类型改为bigint
\n
ALTER TABLE emp2 MODIFY id BIGINT
\n

6、修改字段名

\n    MYSQL里修改字段名的时候需要加上字段的数据类型否则会报错，而CHANGE也可以只修改数据类型，实现和MODIFY同样的效果,方法是将SQL语句中的“新字段名”和“旧字段名”设置为相同的名称，只改变“数据类型”,改变数据类型，例如刚才那个例子，将id列改为bigint数据类型
\n
ALTER TABLE emp2 CHANGE id id BIGINT
\n

7、添加字段

\n
ALTER TABLE emp2 ADD NAME NVARCHAR(200) NULL
\n

8、删除字段

\n    MYSQL删除字段不需要添加COLUMN关键字的
\n
 ALTER TABLE emp2 DROP NAME
\n

9、删除外键约束

\n如果是外键约束，需要使用 DROP FOREIGN KEY，如果是主键约束需要使用DROP PRIMARY KEY
\n    --删除外键约束
\n
 ALTER TABLE emp2 DROP FOREIGN KEY fk_emp_dept
    --删除主键约束
\n
 ALTER TABLE emp2 DROP PRIMARY KEY pk_emp_dept
\n

10、删除表

\n
DROP TABLE emp2
但是如果要同时删除多个表或者删除之前要先判断一下
\n
DROP TABLE IF EXISTS emp1 ,emp2
