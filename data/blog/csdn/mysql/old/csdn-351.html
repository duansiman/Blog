\n

\n

语法

\n

\n
INSERT INTO table_name1(column_list1) SELECT (column_list2) FROM table_name2 WHERE (condition)
\n

\n

table_name1指定待插入数据的表；column_list1指定待插入表中要插入数据的哪些列；table_name2指定插入数据是从

\n

哪个表中查询出来的；column_list2指定数据来源表的查询列，该列表必须和column_list1列表中的字段个数相同，数据类型相同；

\n

condition指定SELECT语句的查询条件

MYSQL和SQLSERVER的区别：

\n

区别一

\n

当要导入的数据中有重复值的时候，MYSQL会有三种方案

\n

方案一：使用 ignore 关键字
\n方案二：使用 replace into
\n方案三：ON DUPLICATE KEY UPDATE

\n

第二和第三种方案这里不作介绍，因为比较复杂，而且不符合要求，这里只讲第一种方案

\n

\n
TRUNCATE TABLE person\n\nTRUNCATE TABLE persona_old \n\nINSERT INTO person_old\nVALUES (11,'Harry',20,'student'),(12,'Beckham',31,'police')\n\n##注意下面这条insert语句是没有ignore关键字的\nINSERT INTO person(id,NAME,age,info)\nSELECT id,NAME,age,info FROM person_old;\n\nINSERT INTO person_old \nVALUES (13,'kay',26,'student')\n\n##注意下面这条insert语句是有ignore关键字的\nINSERT IGNORE INTO person(id,NAME,age,info)\nSELECT id,NAME,age,info FROM person_old;
\n

\n

 

\n

\n

可以看到插入成功

\n

SQLSERVER

\n

在SQLSERVER这边，如果要忽略重复键，需要在建表的时候指定 WITH (IGNORE_DUP_KEY= \nON) ON \n[PRIMARY] 

\n

这样在插入重复值的时候，SQLSERVER第一次会保留值，第二次发现有重复值的时候，SQLSERVER就会忽略掉

\n

区别二

\n

插入自增列时的区别

\n

SQLSERVER需要使用 SETIDENTITY_INSERT 表名 \nON 才能把自增字段的值插入到表中，如果不加 SET IDENTITY_INSERT 表名 ON 

\n

则在插入数据到表中时，不能指定自增字段的值，则id字段不能指定值，SQLSERVER会自动帮你自动增加一

\n

\n
INSERTINTO person(NAME,age,info) VALUES ('feicy',33,'student')
\n

\n

而MYSQL则不需要，而且自由度非常大

\n

你可以将id字段的值指定为NULL，MYSQL会自动帮你增一

\n
\n
INSERTINTO person(id,NAME,age,info) VALUES (NULL,'feicy',33,'student') \n
\n
\n

\n

也可以指定值

\n

\n
INSERT IGNORE INTO person(id,NAME,age,info) VALUES (16,'tom',88,'student')
\n

\n

\n

也可以不写id的值，MYSQL会自动帮你增一

\n

\n
INSERT IGNORE INTO person(NAME,age,info) VALUES ('amy',12,'bb')
\n

\n

\n

你可以指定id字段的值也可以不指定，指定的时候只要当前id字段列没有你正在插入的那个值就可以，即没有重复值就可以

\n

自由度非常大，而且无须指定 SET IDENTITY_INSERT 表名 ON 选项

\n

区别三

\n

唯一索引的NULL值重复问题

\n

MYSQL

\n

在MYSQL中UNIQUE 索引将会对null字段失效

\n

\n
insert into test(a) values(null)\n \ninsert into test(a) values(null)
\n

\n

上面的插入语句是可以重复插入的（联合唯一索引也一样）

\n

SQLSERVER

\n

SQLSERVER则不行

\n

\n
CREATE TABLE person (\n id INT NOT NULL IDENTITY(1,1),\n NAME CHAR(40) NULL DEFAULT '',\n age INT NOT NULL DEFAULT 0,\n info CHAR(50) NULL,\n PRIMARY KEY (id)\n)\n\nCREATE UNIQUE INDEX IX_person_unique ON [dbo].[person](name)\n\nINSERT INTO [dbo].[person]\n ( [NAME], [age], [info] )\nVALUES ( NULL, -- NAME - char(40)\n 1, -- age - int\n 'aa' -- info - char(50)\n ),\n ( NULL, -- NAME - char(40)\n 2, -- age - int\n 'bb' -- info - char(50)\n )
\n

\n
