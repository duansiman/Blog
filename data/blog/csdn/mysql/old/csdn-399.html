1. 创建表

\n

MySQL中创建表是通过create table语句实现的，语法这里就不介绍了，非常复杂，可以去官网上查询，所幸的是，我们并不需要记住所有的选项，因为大部分都采用默认即可，下面介绍几个比较常用的选项。

\n

1）指定存储引擎

\n

默认的存储引擎由default-storage-engine指定，如果没有指定，则为MyISAM，如果建表时，你不想使用默认的存储引擎，可以通过如下语句实现：

\n

\n
create table table_name ( ... ) engine=engine_name;
2）只有当表不存在时才创建 \n

\n

\n
create table if not exist table_name ... ;
上述语句表示：如果表已存在，则跳过；否则创建新表。 \n

\n

3）创建临时表

\n

MySQL的临时表有如下特性：

\n

  - 临时表只在创建该表的会话里是可见的，对其它会话是隐藏的，会话断开后，临时表自动被删除。

\n

  - 当临时表的表名和普通表一样时，当前会话将会优先访问临时表。

\n

临时表的语法如下：

\n

\n
create temporary table temp_table_name ... ;

2. 删除表

\n

\n

相比于创建表，删除表就简单多了。

\n

1）删除普通表

\n

\n
drop table table_name;
2）删除临时表 \n

\n

\n
drop temporary table table_name;
3）表存在才删除 \n

\n

\n
drop table if exists table_name;

3. 创建索引

\n
你可以使用和Oracle类似的create index语句重建索引，也可以使用MySQL独有的alter table语句添加索引，在MySQL里，建议用alter table语句，因为它更灵活多能，如：
\n
alter table tbl_name add index index_name (index_cols);\nalter table tbl_name add unique index_name(index_cols);\nalter table tbl_name add primary key(index_cols);\nalter table tbl_name add full text index_name(index_cols);\nalter table tbl_name add spatial index_name(index_cols);
其中，primary key和spatial索引要求索引列必须为NOT NULL，index_name可省略，一般不建议省略，如果省略，系统会自动取个名字。
\n
该语句支持在一个alter table语句添加多个索引，这样更灵活。
\n
在创建表时，也可以同时创建索引，如：
\n
create table tbl_name\n(\n ...\n index index_name (index_cols),\n primary key (index_cols),\n unique (index_cols),\n ...\n);

\n

4. 删除索引

\n
\n
和创建索引类似，删除索引可以用drop index或alter table语句完成，如：
\n
drop index index_name on tbl_name;\ndrop index `primary` on tbl_name;\n\nalter table tbl_name drop index index_name;\nalter table tbl_name drop primary index;

5. 改变表结构

\n
1）改变列的数据类型
\n
如果你想改变列的数据类型，可以使用change或modify字句，如下所示：
\n
alter table tbl_name modify col_name mediumint unsigned;\nalter table tbl_name change old_col_name new_col_name mediumint unsigned;
从上面可以知道，change功能更强大，它不但能够改变列的数据类型，还可以重命名列，但是它比较麻烦的一点是即时不想重命名，也要把列名写两遍。
\n

\n
\n
2）改变列的默认值
\n
MySQL可以通过alter column语句设置或移除列的默认值，该语句直接修改.frm文件，因此速度非常快。
\n
ALTER TABLE MyTable ALTER COLUMN foo SET DEFAULT 'bar';\nALTER TABLE MyTable ALTER COLUMN foo DROP DEFAULT;
\n

\n
\n
3）改变表的存储引擎
\n
alter table tbl_name engine=engine_name;
\n

\n
\n4）重命名表
\n
重命名表可以使用alter table ... rename to 或rename table，如：
\n
alter table tbl_name rename to new_tbl_name;\nrename table tbl_name to new_tbl_name;
alter table ... rename每次只能重命名一张表，而rename table可以重命名多张表，如：
\n
rename table t1 to new_t1, t2 to t1;
如果在表名前加上前缀，可以把一个表从一个数据库迁移到另一个数据库。
\n
