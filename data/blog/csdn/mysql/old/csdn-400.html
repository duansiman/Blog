MySQL初始账户管理

\n

\n
\n
MySQL的初始账户如下：
\n
[root@lx16 ~]# mysql -u root\nmysql> select host,user,password from mysql.user;\n+-----------+------+----------+\n| host | user | password |\n+-----------+------+----------+\n| lx16 | root | |\n| 127.0.0.1 | root | |\n| ::1 | root | |\n| localhost | | |\n| lx16 | | |\n| localhost | root | |\n+-----------+------+----------+
MySQL有两类初始用户：
\n
\n
    \n
  1. root超级账户：拥有全部的权限，可以做任何事。
  2. 匿名账户：如何人都可以通过它连接服务器，但它权限很小。
\n
在默认情况下，这些账户都没有口令，因此为了安全起见，我们首先必须得为所有的root账户设置密码。
\n
设置密码的第一种方法是用SET PASSWORD语句，假如我们现在要给'root'@'localhost'设置口令，只要执行：
\n
mysql> set password for 'root'@'localhost'=password('*****');
\n

\n
\n设置密码的第二种方法是直接update user权限表，这种方法的好处是可以同时给多个账户设置密码，如下面的语句可以一次修改所有root账户的密码：
\n
mysql> update mysql.user set password=password('***') where user='root';\nmysql> flush privileges;
如果用update方式修改，必须明确告诉服务器重新加载权限表(flush privileges)
\n

\n
\n
对于匿名账户，强烈建议将他们删除，删除语句如下：
\n
mysql> drop user ''@'localhost';\nmysql> drop user ''@'lx16';

\n执行完以上操作之后，user权限表里的数据如下：
\n
mysql> select host,user,password from mysql.user;\n+-----------+------+-------------------------------------------+\n| host | user | password |\n+-----------+------+-------------------------------------------+\n| lx16 | root | *578EC7851088AC1F2A67B100540344B03BD2BA99 |\n| 127.0.0.1 | root | *578EC7851088AC1F2A67B100540344B03BD2BA99 |\n| ::1 | root | *578EC7851088AC1F2A67B100540344B03BD2BA99 |\n| localhost | root | *578EC7851088AC1F2A67B100540344B03BD2BA99 |\n+-----------+------+-------------------------------------------+

创建新账户

\n

\n
\nMySQL不仅要求你必须知道谁(user_name)能连接，还必须指定从什么地方连接(host_name)，也就是说即便两个账户拥有相同的名字，如果他们将从不同客户端连接，你也要为它们各自创建一个账户。
\n
可以利用以下两个通配符灵活配置主机名的限制：
\n
\n
    \n
  1. ‘%’ - 匹配任何多个字符
  2. ‘-’ - 配置一个字符
\n
test账户可以从任意IP连接\ncreate user 'test'@'%' identified by '***';\ntest账户只能从本地连接\ncreate user 'test'@'localhost' identified by '***';\ntest账户只能从'192.168.2.%'网段连接\ncreate user 'test'@'192.168.2.%' identified by '***';\n还可以使用IP掩码\ncreate user 'test'@'192.168.2.2

用户资源限制

\n

\n
\n
通过Grant的with字句限制单个用户的使用资源：
\n
MAX_QUERIES_PER_HOURS 限制用户每小时执行查询的最大数，\nMAX_UPDATES_PER_HOUR在UPDATE 限制用户每小时执行的最大更新数。\nMAX_CONNECTIONS_PER_HOUR 限制用户每小时可连接的最大数。\nMAX_USER_CONNECTIONS 限制用户连接的最大并发数
下面是一个例子：
\n
\n
\n
GRANT ALL PRIVILEGES ON baby.* TO baby@'10.10.100.%' IDENTIFIED BY '***' \nWITH MAX_CONNECTIONS_PER_HOUR 10 \nMAX_USER_CONNECTIONS 10;

\n
\n
grant select, update (street,city) on sampdb.member to 'test'@'localhost';
\n \n
