在我们建好表之后，我们会发现，经常需要比如增加字段，删除字段，修改表名等等等等一系列关于表结构的操作，那么接下来干货来啦！

Alter命令，被我们常用来修改表结构：
\n1）现在我想要给新建好的test数据库的table04表格新增加一个字段怎么做呢？
\n首先进入到test数据库目录下面，
\n回顾一下，输入命令use test;
\n然后开始增加(使用add 关键字)字段操作：
\n语法：alter table 表名 add 字段名 字段类型；
\nCase: alter table table04 add sex varchar(10) default ‘unknown’;
\n

\n\n

2)修改列：
\n语法：alter table 表名 modify 字段名 字段类型
\nCase: alter table table04 modify sex varchar(8);

\n\n

修改列名：用change关键字
\n
\n将table04表的address字段改为dizhi,类型为varchar(50)
\nCase：alter table table04 change address dizhi varchar(50);
\n改完的效果：
\n

\n\n

3)给表格改名：
\n语法：alter table 表名 rename 新的表名；
\nCase：alter table table04 rename table_new;
\n表改完名字的效果图：
\n

\n\n

4）删除列：
\n语法：alter table 表名 drop 列名；
\nCase: alter table test04 drop school;
\n

\n\n

—-alter小结—-
\n我们修改表结构，总会用到alter table,它只告诉Mysql我们要来修改了，具体的操作是什么，再加上具体的操作关键字就组成了一个完整的修改sql

\n\n

5)删除表：drop
\n语法：drop table 表名
\n所以删除table03表格，就写成：
\ndrop table table03(这种写法不太完善);
\n但是这样写的话容易产生一个错误，就是当表不存在的情况下，会报错，所以，我们通常会在sql加上if exists 进行避免报错的判断；
\nCase: drop table if exists table03;
