\n六，约束
\n1. 约束保证数据的完整性和一致性。
\n2. 约束分为表级约约束和列级约束。
\n3. 约束类型包括：
\nNOT NULL(非空约束)
\nPRIMARY KEY(主键约束)
\nUNIQUE KEY(唯一约束)
\nDEFAULT(默认约束)
\nFOREIGN KEY(外键约束)
\n
\n
\nFOREIGN KEY（外键约束）
\n作用：
\n保持数据一致性，完整性
\n实现一对一或者一对多关系
\n要求：
\n1. 父表和子表必须使用相同的存储引擎，而且禁止使用临时表。
\n子表： 具有外键列的表。
\n父表： 子表参照的表。
\n2. 数据表的存储引擎只能为InnoDB。
\n3. 外键列和参照列必须具有相似的数据类型。其中数字的长度或是否有符号位必须相同；而字符的长度则可以不同。
\n* 数字长度或是否有符号位必须相同，否则将提示错误并无法创建外键约束： 
\n外键列： 加过外键列的列。
\n参照列： 外键列所参照的列。
\n4. 外键列和参照列必须创建索引。如果外键列不存在索引的话，MySQL将自动创建索引。
\n
\n
\n编辑数据表的默认存储引擎
\nMySQL配置文件： default-storage-engine=INNODB
\n查看创建表的存储引擎: SHOW CREATE TABLE table_name;
\n
\n
\n例子：
\n父表：
\nmysql> CREATE TABLE provinces(
\nid SMALLINT UNSIGNED PRIMARY KEY AUTO_INCREMENT, //参照列： id
\npnane VARCHAR(20) NOT NULL
\n)；
\n子表：
\nmysql> CREATE TABLE users(
\nid SMALLINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
\nusername VARCHAR(20) NOT NULL,
\npid SMALLINT UNSIGNED, 
\nFOREIGN KEY (pid) REFERENCES provinces (id) //外键列： pid 参照列：provinces.id
\n);
\n

七，约束扩展
\n
1.自动编号 - AUTO_INCREMENT
\n自动编号，且必须与主键配合使用
\n自动编号AUTO_INCREMENT
\n1、自动编号：保证记录的唯一性
\n2、类型必须为整型（可以是FLOAT(5,0)等，小数点后必须为0），必须和主键PRIMARY KEY组合使用
\n3、默认情况下，起始值为1，每次的增量为1
\n默认情况下，起始值为1， 每次增量为1.
\nCREATE TABLE tb3(
\nid SMALLINT UNSIGNED AUTO_INCREMENT,
\nusername VARCHAR(30) NOT NULL); //有错，自动增量字段必须设置成主键
\n
\n
\n主键可以写为PRIMARY KEY，也可以写成KEY
\n每张数据表只能存在一个主键
\n主键保证记录的唯一性
\n主键自动为NOT NULL
\nauto_increment必须和主键primary key一起使用，但是主键primary key不一定要和auto_increment一块使用

\n八，外键约束的参照操作：
\n1. CASCADE: 从父表删除或更新且自动删除或更新子表中匹配的行。
\n2. SET NULL: 从父表删除或更新行，并设置子表中的外键列为NULL。如果使用该选项，必须保证子表列没有指定NOT NULL。
\n3. RESTRICT: 拒绝对父表的删除或者更新操作。
\n4. NO ACTION: 标准SQL的关键字，在MySQL中与RESTRICT相同。
\n
\n
\n例子： 
\n添加父表：
\nmysql> CREATE TABLE provinces(
\nid SMALLINT UNSIGNED PRIMARY KEY AUTO_INCREMENT, //参照列： id
\npnane VARCHAR(20) NOT NULL
\n)；
\n添加子表带CASCADE操作：
\nmysql> CREATE TABLE user1(
\nid SMALLINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
\nusername VARCHAR(10) NOT NULL,
\npid SMALLINT UNSIGNED,
\nFOREIGN KEY (pid) REFERENCES provinces (id) ON DELETE CASCADE
\n);
\n为父表添加数据：
\nINSERT INTO provinces(pname) VALUES('A')
\nINSERT INTO provinces(pname) VALUES('B')
\nINSERT INTO provinces(pname) VALUES('C')
\n删除父表中的数据，并影响子表数据：
\nDELETE FROM provinces where id = 3;
\n==> 父表id为3的数据被删， 其子表中pid为3的数据也被删！
\n*外键约束 很少使用物理外键约束（innodb支持），在实际开发用的比较多是逻辑外键（不使用freign key）

\n九，表级约束与列级约束
\n1.表级约束与列级约束
\n（1）对一个数据列建立的约束，称为列级约束
\n（2）对多个数据列建立的约束，称为表级约束
\n（3）列级约束既可以在列定义时声明，也可以在列定以后声明
\n（4）表级约束只能在列定义后声明
\n2.主键、外键、非空、唯一、check，default都是约束条件。主键、外键、既可以作为表级约束，也可作为列级约束
\n而像非空、唯一等（ not null， default ）只有列级约束

\n十一，修改数据表（一）
\n1，添加
\n（1）添加单列
\nALTER TABLE tbl_name ADD [COLUMN] col_name column_definition [FIRST | AFTER col_name]
\n例如：
\nALTER TABLE users1 ADD age TINYINT UNSIGNED NOT NULL DEFAULT 10;
\n在 id 后面添加 username属性：
\nALTER TABLE users1 ADD password VARCHAR(32) NOT NULL AFTER username;
\n（2）添加多列
\nALTER TABLE tbl_name ADD [COLUMN]（col_name column_definition，...）该方法添加的列不可以指定位置
\n2，删除
\n（1）删除单列
\nALTER TABLE tbl_name DROP [COLUMN] col_name
\n例如：ALTER TABLE users1 DROP truename;
\n（2）一次删除多列：
\nALTER TABLE uses1 DROP password,DROP age;
\n也能删除一列的同时，新增一列。其间用逗号隔开。
\n
\n
\n3、添加主键约束：
\nALTER TABLE tbl_name ADD [CONSTRAINT[symbol]] PRIMARY KEY[index_type] (col_name);
\n4、添加唯一约束
\nALTER TABLE tbl_name ADD UNIQUE(col_name);
\n5、添加外键约束
\nALTER TABLE tbl_name ADD [CONSTRAINT[symbol]] FOREIGN KEY [index_type](外键列) REFERENCES 父类（参考列）;
\n6、添加/删除默认DEFAULT约束
\nALTER TABLE tbl_name ALTER col_name SET DEFAULT literal / DROP DEFAULT;
\n
