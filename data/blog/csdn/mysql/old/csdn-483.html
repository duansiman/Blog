MySQL学习笔记—视图

\n\n
\n\n

视图是查看基础表数据的一种方式，其作用有

\n\n

　　- 简化开发难度，可以运用视图执行多表操作
\n　　- 数据安全，开发人员不能直接对表操作，也不能进行删除，修改操作
\n　　- 数据重构，在有限的表中，以不同的角度生成所需的视图，简化业务

\n\n
\n\n

CREATE VIEW语法

\n\n
\n\n

CREATE VIEW语法:

\n\n\n\n
CREATE [OR REPLACE] [ALGORITHM = {UNDEFINED | MERGE | TEMPTABLE}]\n VIEW view_name [(column_list)]\n AS select_statement\n [WITH [CASCADED | LOCAL] CHECK OPTION]
\n\n

该语句能创建新的视图，如果给定了OR REPLACE子句，该语句还能替换已有的视图。select_statement是一种SELECT语句，它给出了视图的定义。该语句可从基表或其他视图进行选择。

\n\n

该语句要求具有针对视图的CREATE VIEW权限，以及针对由SELECT语句选择的每一列上的某些权限。对于在SELECT语句中其他地方使用的列，必须具有SELECT权限。如果还有OR REPLACE子句，必须在视图上具有DROP权限。

\n\n

视图属于数据库。在默认情况下，将在当前数据库创建新视图。要想在给定数据库中明确创建视图，创建时，应将名称指定为

\n\n
db_name.view_name。\n
\n\n

表和视图共享数据库中相同的名称空间，因此，数据库不能包含具有相同名称的表和视图。实际上视图是一个虚表。

\n\n

视图必须具有唯一的列名，不得有重复，就像基表那样。默认情况下，由SELECT语句检索的列名将用作视图列名。要想为视图列定义明确的名称，可使用可选的column_list子句，列出由逗号隔开的ID。column_list中的名称数目必须等于SELECT语句检索的列数。

\n\n

SELECT语句检索的列可以是对表列的简单引用。也可以是使用函数、常量值、操作符等的表达式。

\n\n

对于SELECT语句中不合格的表或视图，将根据默认的数据库进行解释。通过用恰当的数据库名称限定表或视图名，视图能够引用表或其他数据库中的视图。

\n\n

\n\n

可选的ALGORITHM子句是对标准SQL的MySQL扩展。ALGORITHM可取三个值：
\n　　- MERGE
\n　　- TEMPTABLE
\n　　- UNDEFINED

\n\n

如果没有ALGORITHM子句，默认算法是UNDEFINED（未定义的）。算法会影响MySQL处理视图的方式。

\n\n
    \n

  • 对于MERGE，会将引用视图的语句的文本与视图定义合并起来，使得视图定义的某一部分取代语句的对应部分。

    • \n

  • 对于TEMPTABLE，视图的结果将被置于临时表中，然后使用它执行语句。

    • \n

  • 对于UNDEFINED，MySQL将选择所要使用的算法。如果可能，它倾向于MERGE而不是TEMPTABLE，这是因为MERGE通常更有效，而且如果使用了临时表，视图是不可更新的。

    • \n
\n\n

选择TEMPTABLE有一个好处，在创建临时表之后、并在完成语句处理之前，能够释放基表上的锁定。与MERGE算法相比，锁定释放的速度更快，这样，使用视图的其他客户端不会被屏蔽过长时间。

\n\n

有三种情况视图算法是UNDEFINED：

\n\n

　　· 在CREATE VIEW语句中没有ALGORITHM子句。

\n\n

　　· CREATE VIEW语句有1个显式ALGORITHM = UNDEFINED子句。

\n\n

　　· 为仅能用临时表处理的视图指定ALGORITHM = MERGE。在这种情况下，MySQL将生成告警，并将算法设置为UNDEFINED。

\n\n

MERGE算法要求视图中的行和基表中的行具有一对一的关系。如果不具有该关系。必须使用临时表取而代之。如果视图包含下述结构中的任何一种，将失去一对一的关系：

\n\n

　　· 聚合函数（SUM(), MIN(), MAX(), COUNT()等）。

\n\n

　　· DISTINCT

\n\n

　　· GROUP BY

\n\n

　　· HAVING

\n\n

　　· UNION或UNION ALL

\n\n

　　· 仅引用文字值（在该情况下，没有基本表）。如create view v as select 1 as num;

\n\n

某些视图是可更新的。也就是说，可以在诸如UPDATE、DELETE或INSERT等语句中使用它们，以更新基表的内容。对于可更新的视图，在视图中的行和基表中的行之间必须具有一对一的关系。即符合MERGE算法的要求。

\n\n

如果视图包含下述结构中的任何一种，那么它就是不可更新的：

\n\n

　　· 聚合函数（SUM(), MIN(), MAX(), COUNT()等）。

\n\n

　　· DISTINCT

\n\n

　　· GROUP BY

\n\n

　　· HAVING

\n\n

　　· UNION或UNION ALL

\n\n

　　· 位于选择列表中的子查询

\n\n

　　· Join

\n\n

　　· FROM子句中的不可更新视图

\n\n

　　· WHERE子句中的子查询，引用FROM子句中的表。

\n\n

　　· 仅引用文字值（在该情况下，没有要更新的基本表）。

\n\n

　　· ALGORITHM = TEMPTABLE（使用临时表总会使视图成为不可更新的）。

DROP VIEW能够删除1个或多个视图。必须在每个视图上拥有DROP权限。

\n\n

可以使用关键字IF EXISTS来防止因不存在的视图而出错。
