\n

为了连接服务器，当调用mysql时，你通常需要提供一个用户名并且很有可能需要一个密码。如果服务器运行在登录服务器之外的其它机器上，就需要你指定一个主机名。联系管理员以找出连接时需要用到哪些联接参数（即，联接的主机、用户名和使用的密码）。一旦你知道了正确的参数，就可以按照以下的方式进行连接。

\n\n
shell>mysql -h host -u user -p\nEnter password: ******
\n\n

host和user分别代表MySQL运行的主机名和mysql账户用户名。设置时替换成合适的值。****代表你的密码；当mysql显示Enter password:时，输入它。

\n\n

如果有效，你应该看见mysql>提示符后面的一些介绍信息。

\n\n\n\n
shell> mysql -h host-u user-p\nEnter password: ********\nWelcome to the MySQL monitor. Commands end with ; or \\g.\nYour MySQL connection id is 25338 to server version: 5.6.29-standard\n\nType 'help;' or '\\h' for help. Type '\\c' to clear the buffer.\nmysql>
\n\n

mysql提示符告诉你mysql已经准备好为你输入SQL 语句。

\n\n

如果你登陆的和mysql运行的是同一个机器，你可以忽略主机名
\n，直接使用下列的语句：

\n\n\n\n
shell> mysql -u user-p
\n\n

如果当你尝试登陆的时候，碰到一个像“ERROR 2002 (HY000): Can’t
\nconnect to local MySQL server through socket ‘/tmp/mysql.sock’ (2)”的错误信息，这意味着你的mysql 服务器没有运行。这时，你需要联系管理员或者看 与你的操作系统相关的Chapter 2, Installing and Upgrading MySQL 部分内容。

\n\n

为了帮助您登陆时遇到的其它问题，请查看Section B.5.2, “Common Errors When Using MySQL Programs”.

\n\n

一些MySQL安装允许用户以匿名（未命名）用户连接到本地主机上运行的服务器。如果你的机器是这种情况，你应该能不带任何选项地调用mysql与该服务器连接；

\n\n
shell> mysql
\n\n

成功地连接后，可以在mysql>提示下输入QUIT (或\\q)随时退出

\n\n\n\n
mysql> QUIT\nBye
\n\n

在Unix中，你还可以按ctrl +D 键断开连接。
\n下面的很多的例子都假设你连接了服务器，由mysql>提示符指明。
