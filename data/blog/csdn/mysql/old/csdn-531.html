\n \n
本篇介绍了MySQL中的各种约束，怎么使用这些约束，如何使用limit进行分页
\n

\n
\n

约束

\n

\n
\n \n \n \n \n \n \n
\n
为了保证数据的一致性和完整性，SQL规范以约束的方式对表数据进行额外的条件限制。
\n
约束是表级的强制规定 可以在创建表时规定约束（通过 CREATE TABLE 语句），或者在表创建之后也可以（通过ALTERTABLE 语句 ）
\n
\n

\n
\n

常用约束

\n
\n \n \n \n \n \n \n
\n
①not null (非空), 只能用于列级约束
\n
②unique(唯一)
\n
③primary key(主键--非空且唯一) 表中只允许一个主键
\n
④foreign key(外键--引用别的表的数据)
\n
⑤check, 只能用于列级约束
\n
⑥default 缺省值
\n
\n

\n
\n
注意： MySQL不支持check约束，但可以使用check约束，而没有任何效果；
\n

\n
\n

约束分类

\n

\n
\n \n \n \n \n \n \n \n \n \n
\n
根据约束数据列的限制：
\n
单列约束：每个约束只约束一列
\n
多列约束：每个约束可约束多列数据
\n
\n
根据约束的作用范围
\n
列级约束只能作用在一个列上，跟在列的定义后面
\n
表级约束可以作用在多个列上，不与列一起，而是单独定义
\n
\n

\n
\n

NOT NULL 约束

\n

\n
\n \n \n \n \n \n \n
\n
非空约束用于确保当前列的值不为空值，非空约束只能出现在表对象的列上。
\n
Null类型特征：
\n
      所有的类型的值都可以是null，包括int、float等数据类型
\n
      空字符串""不等于null，0也不等于null
\n
\n

\n
\n

\n
\n \n \n \n \n \n \n
\n
NOT NULL约束使用:
\n
①直接在目标列定义后添加NOT NULL
\n
②ALTER TABLE 表名
\n
        MODIFY  列定义
\n
③使用MODIFY还可以取消列约束
\n
\n

\n
\n
创建NOT NULL 约束
\n
\n
CREATE TABLE emp(
\n
    id INT(10) NOT NULL,
\n
    NAME VARCHAR(20) NOT NULL DEFAULT 'abc',
\n
    sex CHAR NULL
\n
);
\n
\n

\n
\n
增加NOT NULL约束
\n
\n
ALTER TABLE emp
\n
    MODIFY sex VARCHAR(30) NOT NULL;
\n
\n

\n
\n
取消NOT NULL约束
\n
\n
ALTER TABLE emp
\n
    MODIFY sex VARCHAR(30) NULL;
\n
\n

\n
\n
取消NOT NULL约束，并添加默认值
\n
\n
ALTER TABLE emp
\n
    MODIFY NAME VARCHAR(15) \nDEFAULT 'abc' NULL;
\n
\n

\n

UNIQUE 约束

\n

\n
\n \n \n \n \n \n \n
\n唯一约束，值不能重复，但是允许出现多个空值：NULL。
\n

\n
\n \n \n \n \n \n \n
\n
同一个表可以有多个唯一约束，多个列组合的约束。
\n
在创建唯一约束的时候，如果不给唯一约束名称，就默认和列名相同。
\n
MySQL会给唯一约束的列上默认创建一个唯一索引
\n
\n

\n
\n
使用表级约束，设置唯一约束。用户名和密码不能重复
\n
\n
\n
CREATE TABLE USER(
\n
 id INT NOT NULL,
\n
 NAME VARCHAR(25),
\n
 PASSWORD VARCHAR(16),
\n
 #使用表级约束语法
\n
 CONSTRAINT uk_name_pwd UNIQUE(NAME,PASSWORD)
\n
);
\n
\n

\n
\n \n \n \n \n \n \n
\n
表级约束，在列定义完后，使用关键字进行定义
\n
语法：约束关键字(列名)   CONSTRAINT uk_name_pwd 可以省略
\n
\n

\n
\n
在非定义的时候给某表添加,丢弃表级的唯一约束
\n

\n
\n \n \n \n \n \n \n
\n
添加
\n
alter table 表名
\n
    add unique(列名);  
\n
\n

\n
\n
添加唯一约束
\n
\n
ALTER TABLE USER
\n
    ADD UNIQUE(NAME,PASSWORD);
\n
-------------------------------
\n
ALTER TABLE USER
\n
    ADD CONSTRAINT uk_name_pwd UNIQUE(NAME,PASSWORD);
\n
-------------------------------
\n
ALTER TABLE USER
\n
    MODIFY NAME VARCHAR(20) UNIQUE;
\n
\n

\n
\n \n \n \n \n \n \n
\n
删除
\n
alter table 表名
\n
    DROP index  ‘列名’
\n
\n

\n
\n
删除唯一约束
\n
\n
ALTER TABLE USER
\n
    DROP INDEX uk_name_pwd;
\n
-----------------------------
\n
ALTER TABLE USER
\n
    DROP INDEX 'NAME,PASSWORD';
\n

PRIMARY KEY 约束

\n

\n
\n \n \n \n \n \n \n
\n
主键约束相当于唯一约束+非空约束的组合，主键约束列不允许重复，也不允许出现空值
\n
如果是多列组合的主键约束，那么这些列都不允许为空值，并且组合的值不允许重复。
\n
每个表最多只允许一个主键，建立主键约束可以在列级别创建，也可以在表级别上创建。
\n
MySQL的主键名总是PRIMARY，当创建主键约束时，系统默认会在所在的列和列组合上建立对应的唯一索引。
\n
\n

\n
\n
添加主键约束
\n
列级
\n
\n
CREATE TABLE emp4(
\n
    id INT AUTO_INCREMENT PRIMARY KEY,
\n
    NAME VARCHAR(20)
\n
);
\n
\n

\n
\n
表级
\n
\n
CREATE TABLE emp5(
\n
    id INT NOT NULL AUTO_INCREMENT,
\n
    NAME VARCHAR(20),
\n
    pwd VARCHAR(15),
\n
    CONSTRAINT emp5_id_pk PRIMARY KEY(id)
\n
);
\n
\n

\n
\n
组合
\n
\n
CREATE TABLE emp6(
\n
    id INT NOT NULL,
\n
    NAME VARCHAR(20),
\n
    pwd VARCHAR(15),
\n
    CONSTRAINT emp7_pk PRIMARY KEY(NAME,pwd)
\n
);
\n
\n

\n
\n \n \n \n \n \n \n
\n定义表级约束的时候 CONSTRAINT emp7_pk可以省略
\n

\n
\n
删除主键约束
\n
\n
ALTER TABLE emp5
\n
    DROP PRIMARY KEY;
\n
\n

\n
\n
添加主键约束(表建立后添加)
\n
\n
ALTER TABLE emp5
\n
     ADD PRIMARY KEY(NAME,pwd);
\n
\n

\n
\n
修改主键约束
\n
\n
ALTER TABLE emp5
\n
     MODIFY id INT PRIMARY KEY;
\n

FOREIGN KEY 约束

\n \n \n \n \n \n \n
\n
外键约束是保证一个或两个表之间的参照完整性，外键是构建于一个表的两个字段或是两个表的两个字段之间的参照关系。
\n
从表的外键值必须在主表中能找到或者为空。当主表的记录被从表参照时，主表的记录将不允许删除，如果要删除数据，需要先删除从表中依赖该记录的数据，然后才可以删除主表的数据。 还有一种就是级联删除子表数据。
\n
注意：外键约束的参照列，在主表中引用的只能是主键或唯一键约束的列
\n
同一个表可以有多个外键约束
\n
\n

\n
\n \n \n \n \n \n \n
\n
外键创建方式：
\n
[constraint 外键名] foreign key(子表的引用列) references 母表(母表的主键);
\n
\n

\n
\n
创建外键约束
\n
\n
CREATE TABLE dept(
\n
    dept_id INT AUTO_INCREMENT PRIMARY KEY,
\n
    dept_name VARCHAR(20)
\n
);                                             主表
\n
------------------------------------------
\n
CREATE TABLE emp(
\n
    emp_id INT AUTO_INCREMENT PRIMARY KEY,
\n
    last_name VARCHAR(15),
\n
    dept_id INT,
\n
    CONSTRAINT emp_dept_id_fk FOREIGN KEY(dept_id)
\n
    REFERENCES dept(dept_id)
\n
);                                             从表
\n
\n

\n
\n
创建多列外键组合，必须使用表级约束
\n
\n
CREATE TABLE classes(
\n
    id INT,
\n
    NAME VARCHAR(20),
\n
    number INT,
\n
    PRIMARY KEY(NAME,number)
\n
);                                             主表
\n
------------------------------------------    
\n
CREATE TABLE student(
\n
    id INT AUTO_INCREMENT PRIMARY KEY,
\n
    classes_name VARCHAR(20),
\n
    classes_number INT,
\n
    FOREIGN KEY(classes_name,classes_number)
\n
    REFERENCES classes(NAME,number)
\n
);                                             从表 
\n
\n

\n
\n \n \n \n \n \n \n
\n 当主键约束为两个列时只有两个列连接值一样的时候才会违反约束。
\n

\n
\n

\n
\n \n \n \n \n \n \n
\n
在非定义的时候添加,丢弃外键约束语法
\n alter table 表名
\n add [constraint stu_fk] foreign key(classid) references class(id);
\n 
\n alter table 表名
\n
 drop foreign key 外键名;
\n
\n

\n
\n
删除外键约束
\n
\n
ALTER TABLE emp
\n
    DROP FOREIGN KEY emp_dept_id_fk;
\n
\n

\n
\n
增加外键约束
\n
\n
ALTER TABLE emp
\n
    ADD [CONSTRAINT emp_dept_id_fk] FOREIGN KEY(dept_id)
\n
    REFERENCES dept(dept_id);
\n
\n

\n
\n
FOREIGN KEY 约束的关键字
\n

\n
\n \n \n \n \n \n \n
\n
FOREIGN KEY: 在表级指定子表中的列
\n
REFERENCES: 标示在父表中的列
\n
ON DELETE CASCADE(级联删除): 当父表中的列被删除时，子表中相对应的列也被删除
\n
ON DELETE SET NULL(级联置空): 子表中相应的列置空
\n
\n

\n
\n

\n
\n \n \n \n \n \n \n
\n
设置级联删除
\n
语法：
\n
设置外键，在删除母表记录时，子表记录会被级联，也被删除
\n
alter table 表名
\n
   add [constraint 键名] foreign key(子列引用的列) references 母表(母表主键) on delete cascade on update cascade;
\n
  
\n
设置外键，在删除母表记录时，子表记录中的引用列被置为null
\n
alter table 表名
\n
    add [constraint 键名] foreign key(子列引用的列) references 母表(母表主键) on delete set null on update set null;
\n
\n

\n
\n
\n
级联删除
\n
alter table classes
\n
    add constraint my_fk foreign key(master) references teachers(id) on delete cascade on update cascade;
\n
级联置null
\n
alter table classes
\n
    add constraint my_fk foreign key(master) references teachers(id) on delete set null on update set null;
\n
\n

\n
\n
级联删除设置
\n
\n
CREATE TABLE student(
\n
    id INT AUTO_INCREMENT PRIMARY?KEY,
\n
    NAME VARCHAR(20),
\n
    classes_name VARCHAR(20),
\n
    classes_number INT,
\n
    /*表级别联合外键*/
\n
    FOREIGN?KEY(classes_name, classes_number)
\n
    REFERENCES classes(NAME, number) ON DELETE CASCADE);
\n

CHECK 约束

\n

\n
\n \n \n \n \n \n \n
\nMySQL可以使用check约束，但check约束对数据验证没有任何作用,添加数据时，没有任何错误或警告
\n

\n
\n
\n
CREATE TABLE temp(
\n
    id INT AUTO_INCREMENT,
\n
    NAME VARCHAR(20),
\n
    age INT CHECK(age > 20),
\n
    PRIMARY KEY(id)
\n
);
\n
