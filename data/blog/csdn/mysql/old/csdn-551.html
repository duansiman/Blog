1 简介    

\n    MySQL 的 Query Profiler 是一个使用非常方便的 Query 诊断分析工具，通过该工具可以获取一条Query 在整个执行过程中多种资源的消耗情况，如 CPU，IO，IPC，SWAP 等，以及发生的 PAGE FAULTS，CONTEXT SWITCHE 等等，同时还能得到该 Query 执行过程中 MySQL 所调用的各个函数在源文件中的位置。
\n    
\n    MySQL5.0.37版本以上支持PROFILING调试功能，让您可以了解SQL语句消耗资源的详细信息。因为它需要调用系统的getrusage()函数，所以只是在Linux/Unix类平台上才能使用，而不能在Windows平台上使用。而且，PROFILING是针对处理进程(process)而不是线程(thread)的，服务器上的其他应用，可能会影响您的调试结果，因此，这个工具适合开发过程中的调试，如果要在生产环境中调试使用，则要注意它的局限性。
\n

2 操作

\n2.1 查看是否已经启用profile，默认是关闭的。
\n
mysql> select @@profiling;\n+-------------+\n| @@profiling |\n+-------------+\n| 0 | \n+-------------+\n1 row in set (0.00 sec)

\n
\n2.2 启用profiling。变量profiling是用户变量，每次都得重新启用。
\n
mysql> set profiling = 1;  \nQuery OK, 0 rows affected (0.00 sec)\n\n\nmysql> select @@profiling;  \n+-------------+\n| @@profiling |\n+-------------+\n|           1 | \n+-------------+\n1 row in set (0.00 sec)\n

\n
\n

2.3 执行以下语句。为避免之前已经把 SQL 存放在 QCACHE 中， 建议在执行 SQL 时， 强制 SELECT 语句不进行 QCACHE 检测。这样可以提交分析的准确性。

\n

\n
mysql> use db_user;\nmysql> select sql_no_cache count(*) from system_user;\nmysql> update system_user set username = 'CleverCode' where id = 10;\nmysql> select sql_no_cache count(*) from system_user where age > 20;\n......

\n2.4 使用show profile查询最近一条语句的执行信息。（分析：select sql_no_cache count(*) from system_user where age > 20） \n

\n

\n
mysql> show profile;

\n
\n

\n


\n

\n

2.5 使用show profiles。查看在服务器上执行语句的列表。(查询id，花费时间，语句) 。
\n

\n
mysql> show profiles;
\n

mysql> show profile for query 6;
