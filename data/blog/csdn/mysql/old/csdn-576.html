“使用”一个数据库，使其作为当前数据库
\n命名规则
\n 数据库名不得超过30个字符，变量名限制为29个
\n 必须只能包含 A–Z, a–z, 0–9, _共63个字符
\n 不能在对象名的字符间留空格
\n 必须不能和用户定义的其他对象重名
\n 必须保证你的字段没有和保留字、数据库系统或常用方法冲突
\n 保持字段名和类型的一致性,在命名字段并为其指定数据类型的时候一定要保证一致性。假如数据类型在一个表里是整数,那在另一个表里可就别变成字符型了
\nCREATE TABLE 语句
\n 必须具备:
\n CREATE TABLE权限
\n 存储空间
\n 必须指定:
\n 表名
\n 列名, 数据类型, 尺寸
\n CREATE TABLE [schema.]table
\n (column datatype [DEFAULT expr][, …]);
\n创建表
\n 语法
\n CREATE TABLE dept
\n (deptno INT(2),
\n dname VARCHAR(14),
\n loc VARCHAR(13));
\n 确认
\n DESCRIBE dept
\n常用数据类型
\n INT 使用4个字节保存整数数据
\n CHAR(size) 定长字符数据。若未指定，默认为1个字符，最大长度255
\n VARCHAR(size) 可变长字符数据，根据字符串实际长度保存，必须指定长度
\n FLOAT(M,D) 单精度，M=整数位+小数位，D=小数位。 D<=M<=255,0<=D<=30，默认M+D<=6
\n DOUBLE(M,D) 双精度。D<=M<=255,0<=D<=30，默认M+D<=15
\n DATE 日期型数据，格式’YYYY-MM-DD’
\n BLOB 二进制形式的长文本数据，最大可达4G
\n TEXT 长文本数据，最大可达4G

使用子查询创建表
\n使用 AS subquery 选项，将创建表和插入数据结合起来
\n指定的列和子查询中的列要一一对应
\n通过列名和默认值定义列

\n\n\n\n
CREATE TABLE table \n [(column, column...)]\n AS subquery;

\n\n\n\n
CREATE TABLE dept80 \n AS \n SELECT employee_id, last_name, \n salary*12 ANNSAL, hire_date \n FROM employees \n WHERE department_id = 80;

 SELECT employee_id, last_name, job_id, salary\n FROM employees\n WHERE salary < ALL \n (SELECT salary \n FROM employees \n WHERE job_id = 'IT_PROG')\n AND job_id <> 'IT_PROG';\n

数据库事务

\n\n

**
\n 事务：一组逻辑操作单元,使数据从一种状态变换到另一种状态。
\n 数据库事务由以下的部分组成:
\n 一个或多个DML 语句
\n 一个 DDL(Data Definition Language – 数据定义语言) 语句
\n 一个 DCL(Data Control Language – 数据控制语言) 语句
\n 在数据库编程语言中，事务是将一个数据处理执行步骤的集合作为一个单元来处理。
\n 也就是说，执行这些步骤就好像是执行一个单个的命令一样。

\n\n

设置提交状态：SET AUTOCOMMIT = FALSE;
\n 或者显式的执行 start transaction或 begin
\n 以第一个 DML 语句的执行作为开始
\n 以下面的其中之一作为结束:
\n COMMIT 或 ROLLBACK 语句
\n DDL 语句（自动提交）
\n 用户会话正常结束
\n 系统异常终止
\n COMMIT和ROLLBACK语句的优点
\n 使用COMMIT 和 ROLLBACK语句,我们可以:
\n 确保数据完整性。
\n 数据改变被提交之前预览。
\n 将逻辑上相关的操作分组。
\n使用预处理语句
\n MySQL服务器支持预处理语句。当想要执行多个查询，而每个查询之间只有很小的差别时，预处理语句将会非常有用。
\n 例如，可以预备一条语句，然后多次执行它，而每次只是数据值不同。
\n 除了提供执行的方便外，预处理语句还能提高性能。
\n 可以在mysql命令行客户端定义与使用预处理语句来测试与调试程序。

\n\n

用户定义变量
\n 用户可以保存一个值到用户定义的变量中（也称为用户变量），然后在以后执行预处理语句时使用它。
\n 用户变量用@var_name表示。可使用SET语句来设置用户变量：
\n SET @var_name = expr [, @var_name = expr] …
\n 下列示例预备了一条语句：
\n 确定一个给定国家有多少种语言被使用
\n 然后使用用户定义的变量来执行它多次，并显示结果：

\n\n
 PREPARE my_stmt FROM \n '\n SELECT COUNT(*) \n FROM CountryLanguage \n WHERE CountryCode= ?\n ';\n SET @code = 'ESP';\n EXECUTE my_stmt USING @code;\n\n SET @code = 'RUS';\n EXECUTE my_stmt USING @code;\n\n DEALLOCATE PREPARE my_stmt; \n
\n\n

可使用PREPARE语句定义一条SQL语句，用以在以后执行。
\n 语句可以是不完整的，在预备时的未知数据值可以由问号（?）来表示，它作为参数标记。
\n 在语句被执行时，可以为语句的每个参数指定一个数据值。服务器将会用数据值来替换标记符号以完成该语句。
\n 语句在每次执行时可以使用不同的值。
\n 如果PREPARE语句使用的名称已经存在，服务器将丢弃该名称原有的预处理语句，然后预备一个新的语句。
\n释放预处理语句
\n 当预处理语句被重新定义，或与服务器的连接被中断后，预处理语句将被自动删除。
\n 可使用DEALLOCATE PREPARE语句来释放语句：
\n DEALLOCATE PREPARE namepop;
\n MySQL还提供了DROP PREPARE作为DEALLOCATE PREPARE语句的别名。
