什么是约束？

\n为了保证数据的一致性和完整性，SQL规范以约束的方式对表数据进行额外的条件限制。
\n约束是表级的强制规定：
\n可以在创建表时规定约束（通过 CREATE TABLE 语句），或者在表创建之后也可以（通过 ALTER TABLE 语句）
\n有以下六种约束:
\nNOT NULL 非空约束，规定某个字段不能为空
\nUNIQUE  唯一约束，规定某个字段在整个表中是唯一的
\nPRIMARY KEY  主键(非空且唯一)
\nFOREIGN KEY  外键
\nCHECK  检查约束
\nDEFAULT  默认值
\n注意： MySQL不支持check约束，但可以使用check约束，而没有任何效果；
\n根据约束数据列的限制，约束可分为：
\n单列约束：每个约束只约束一列
\n多列约束：每个约束可约束多列数据
\n根据约束的作用范围，约束可分为：
\n列级约束只能作用在一个列上，跟在列的定义后面
\n表级约束可以作用在多个列上，不与列一起，而是单独定义
\n

NOT NULL 约束

\n非空约束用于确保当前列的值不为空值，非空约束只能出现在表对象的列上。
\nNull类型特征：
\n所有的类型的值都可以是null，包括int、float等数据类型
\n空字符串””不等于null，0也不等于null
\n创建 not null 约束： \n
CREATE TABLE emp(\n\tid INT(10) NOT NULL,\n\tNAME VARCHAR(20) NOT NULL DEFAULT 'abc',\n\tsex CHAR NULL\n);
增加 not null 约束：
\n
ALTER TABLE emp\nMODIFY sex VARCHAR(30) NOT NULL;
\n取消 not null 约束：
\n
ALTER TABLE emp\nMODIFY sex VARCHAR(30) NULL;

\n取消 not null 约束，增加默认值：
\n
ALTER TABLE emp\nMODIFY NAME VARCHAR(15) DEFAULT 'abc' NULL;

\n

UNIQUE 约束   （唯一约束，允许出现多个空值：NULL）

\n同一个表可以有多个唯一约束，多个列组合的约束。在创建唯一约束的时候，如果不给唯一约束名称，就默认和列名相同。
\nMySQL会给唯一约束的列上默认创建一个唯一索引
\n
CREATE TABLE USER( \n\tid INT NOT NULL, \n\tNAME VARCHAR(25), \n\tPASSWORD VARCHAR(16), \n\t#使用表级约束语法 \n\tCONSTRAINT uk_name_pwd UNIQUE(NAME,PASSWORD)\n);
\n表示用户名和密码组合不能重复
\n添加唯一约束
\n
ALTER TABLE USER \nADD UNIQUE(NAME,PASSWORD);\n\t\t\nALTER TABLE USER \nADD CONSTRAINT uk_name_pwd UNIQUE(NAME,PASSWORD);\n\t\t\t\nALTER TABLE USER \nMODIFY NAME VARCHAR(20) UNIQUE;
\n删除约束
\n
ALTER TABLE USER \nDROP INDEX uk_name_pwd;
\n
\n

PRIMARY KEY 约束

\n主键约束相当于唯一约束+非空约束的组合，主键约束列不允许重复，也不允许出现空值
\n如果是多列组合的主键约束，那么这些列都不允许为空值，并且组合的值不允许重复。
\n每个表最多只允许一个主键，建立主键约束可以在列级别创建，也可以在表级别上创建。
\nMySQL的主键名总是PRIMARY，当创建主键约束时，系统默认会在所在的列和列组合上建立对应的唯一索引。
\n列级模式
\n
CREATE TABLE emp4(\n\tid INT AUTO_INCREMENT PRIMARY KEY,\n\tNAME VARCHAR(20)\n);\t
\n表级模式
\n
CREATE TABLE emp5(\n\tid INT NOT NULL AUTO_INCREMENT,\n\tNAME VARCHAR(20),\n\tpwd VARCHAR(15),\n\tCONSTRAINT emp5_id_pk PRIMARY KEY(id)\n);

\n组合模式
\n
CREATE TABLE emp6(\n\tid INT NOT NULL,\n\tNAME VARCHAR(20),\n\tpwd VARCHAR(15),\n\tCONSTRAINT emp7_pk PRIMARY KEY(NAME,pwd)\n);

\n删除主键约束
\n
ALTER TABLE emp5\nDROP PRIMARY KEY;
\n
\n添加主键约束
\n
ALTER TABLE emp5\nADD PRIMARY KEY(NAME,pwd);
\n
\n修改主键约束 \n
\n
ALTER TABLE emp5\nMODIFY id INT PRIMARY KEY;\t

\n

FOREIGN KEY 约束

\n外键约束是保证一个或两个表之间的参照完整性，外键是构建于一个表的两个字段或是两个表的两个字段之间的参照关系。
\n从表的外键值必须在主表中能找到或者为空。当主表的记录被从表参照时，主表的记录将不允许删除，如果要删除数据，需要先删除从表中依赖该记录的数据，然后才可以删除主表的数据。
\n还有一种就是级联删除子表数据。
\n注意：外键约束的参照列，在主表中引用的只能是主键或唯一键约束的列
\n同一个表可以有多个外键约束 \n
\n
\n创建外键约束: \n
\n主表
\n
CREATE TABLE dept(\n<span style="white-space:pre">\t</span>dept_id INT AUTO_INCREMENT PRIMARY KEY,\n\tdept_name VARCHAR(20)\n);\t

\n从表
\n
CREATE TABLE emp(\n\temp_id INT AUTO_INCREMENT PRIMARY KEY,\n\tlast_name VARCHAR(15),\n\tdept_id INT,\n\tCONSTRAINT emp_dept_id_fk FOREIGN KEY(dept_id) \n\tREFERENCES dept(dept_id)\n);
\n
\n创建多列外键组合，必须使用表级约束: \n
\n主表
\n
CREATE TABLE classes(\n\tid INT,\n\tNAME VARCHAR(20),\n\tnumber INT,\n\tPRIMARY KEY(NAME,number)\n);

\n从表
\n
CREATE TABLE student(\n\tid INT AUTO_INCREMENT PRIMARY KEY,\n\tclasses_name VARCHAR(20),\n\tclasses_number INT,\n\tFOREIGN KEY(classes_name,classes_number) \n\tREFERENCES classes(NAME,number)\n);\t

\n删除外键约束：
\n
ALTER TABLE emp\nDROP FOREIGN KEY emp_dept_id_fk;\t

\n增加外键约束：
\n
ALTER TABLE emp\nADD [CONSTRAINT emp_dept_id_fk] FOREIGN KEY(dept_id) REFERENCES dept(dept_id);\t

\nFOREIGN KEY 约束的关键字 \n
\nFOREIGN KEY: 在表级指定子表中的列
\nREFERENCES: 标示在父表中的列
\nON DELETE CASCADE(级联删除): 当父表中的列被删除时，子表中相对应的列也被删除
\nON DELETE SET NULL(级联置空): 子表中相应的列置空
\n
CREATE TABLE student( \n\tid INT AUTO_INCREMENT PRIMARY KEY,\n\tNAME VARCHAR(20), \n\tclasses_name VARCHAR(20),\n\tclasses_number INT, \n\t/*表级别联合外键*/ \n\tFOREIGN KEY(classes_name, classes_number) \n\tREFERENCES classes(NAME, number) ON DELETE CASCADE\n);\t

\n

CHECK 约束

\nMySQL可以使用check约束，但check约束对数据验证没有任何作用,添加数据时，没有任何错误或警告 \n
\n
CREATE TABLE temp(\n<span style="white-space:pre">\t</span>id INT AUTO_INCREMENT,\n\tNAME VARCHAR(20),\n\tage INT CHECK(age > 20),\n\tPRIMARY KEY(id)\n);
