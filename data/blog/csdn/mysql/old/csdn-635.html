\n下面来说说两种存储的区别吧：
\n1、两种数据存储的事务机制不同
\nInnoDB支持事务，Myisam不支持，但是在查询方面Myisam的性能略胜一筹
\n2、锁的机制不同
\nInnoDB为行级锁，能更大程度的支持并发操作；Myisam是表级锁
\n3、数据操作方面
\n修改、新增、删除数据使用InnoDB性能更高，大数据量的查询使用Myisam性能更高
\n查询数据库条数时InnoDB不保存表中数据的条数，每次都重新计算；Myisam会保存每个表的总条数，每次都查询时不需重新计算
\n在查询的时候InnoDB会比Myisam加载很多东西，比如：InnoDB会缓存数据块、Myisam只缓存索引块；InnoDB的寻址是通过映射，Myisam是通过offset
\n4、存储方面
\nInnoDB资源基于表空间数据文件和日志文件，大小只限于操作系统文件大小
\nMyisam以文件的形式存储在磁盘上：.frm存储表定义；.MYD数据文件；.MYI索引文件
\n5、字段自增方面
\n对于auto_increment类型的字段，InnoDB中必须包含只有该字段的索引，但是在Myisam中可以和其他字段一起建立联合索引。
\n6、Mysiam支持全文索引（全文索引可以在VARCHAR或者TEXT类型的列上创建），InnoDB不支持
\n7、delete表的时候InnoDB不会重建表，会一行一行的删除

\n\n

InnoDB表的行级锁不是绝对的，例如：
\nupdate table set num=1 where name like “%a%”;
\n对于不确定的扫描范围，InnoDB同样会扫描全表。
\n说到这里我考虑到数据库的读写分离，读库可以考虑使用Myisam引擎存储，写库使用InnoDB引擎存储，对于数据库的主从同步我们后续再讨论。
\n提升InnoDB数据库性能的方法：
\n针对InnoDB数据库影响性能的主要是innodb_flush_log_at_trx_commit这个选项如果设置为1，那么每次插入数据的时候都会自动提交，导致性能急剧下降，设置为0性能明显提高。
