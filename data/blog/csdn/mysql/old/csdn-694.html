windows下mysql免安装配置

\n


\n1、 下载mysql免安装压缩包

\n

  下载mysql-5.6.22-winx64.zip  解压到本地D:\\mysql-5.6.22-winx64

\n


\n2、  修改配置文件
\n  复制my-default.ini为my.ini  文件

\n


\n3、  修改my.ini

\n

 在[mysqld] 组：
\n  basedir = D:\\\\mysql-5.6.22-winx64
\n  datadir = D:\\\\mysql-5.6.22-winx64\\\\data
\n  port = 3306

\n

4、  命令行安装mysql
\n4.1、注册服务(注意以管理员身份操作：在我的电脑中打开“C:\\Windows\\System32”，找到文件“cmd.exe”，右键点击，选择“以管理员身份运行(A),进到目录“D:\\mysql-5.5.24-winx64”，然后运行以下代码)
\n    bin\\mysqld --install mysql-5.6.22-winx64 --defaults-file=d:\\mysql-5.6.22-winx64\\my.ini

\n

4.2、 启动服务(以管理员身份运行)
\n    net start mysql-5.6.22-winx64
\n     --也可右键点击“我的电脑”|“管理”|“服务和应用程序”|“服务”|找到“mysql-5.5.24-winx64”，右键点开，选择启动

\n


\n4.3、命令行登录（把bin目录设到path环境变量中，则可省略“bin\\”，后面同理）
\n   bin\\mysql -u root -p
\n   --密码为空回车

\n

   为root设密码(原密码为空时)：mysqladmin -u root password "1234"
\n   修改root密码：mysqladmin -u root password oldpass "newpass"

\n

4.4、停止服务(以管理员身份运行)
\n   net stop mysql-5.6.22-winx64

\n


\n4.5 删除服务(以管理员身份运行)
\n    bin\\mysqld --remove mysql-5.6.22-winx64

\n

   启动服务不通过注册
\n   bin\\mysqld --console

\n

   停止服务不通过注册服务
\n   bin\\mysqladmin -u root shutdown

\n

7.修改数据表字符集
\nalter table 表名 default character set 'utf8';

\n

8.建库时指定字符集

\nMYSQL 字符集问题

\n


\nMySQL的字符集支持(Character Set Support)有两个方面：
\n      字符集(Character set)和排序方式(Collation)。
\n对于字符集的支持细化到四个层次:
\n      服务器(server)，数据库(database)，数据表(table)和连接(connection)。
\n1.MySQL默认字符集
\nMySQL对于字符集的指定可以细化到一个数据库，一张表，一列，应该用什么字符集。
\n但是，传统的程序在创建数据库和数据表时并没有使用那么复杂的配置，它们用的是默认的配置，那么，默认的配置从何而来呢？     (1)编译MySQL 时，指定了一个默认的字符集，这个字符集是 latin1；
\n     (2)安装MySQL 时，可以在配置文件 (my.ini) 中指定一个默认的的字符集，如果没指定，这个值继承自编译时指定的；
\n     (3)启动mysqld 时，可以在命令行参数中指定一个默认的的字符集，如果没指定，这个值继承自配置文件中的配置,此时 character_set_server 被设定为这个默认的字符集；
\n     (4)当创建一个新的数据库时，除非明确指定，这个数据库的字符集被缺省设定为character_set_server；
\n     (5)当选定了一个数据库时，character_set_database 被设定为这个数据库默认的字符集；
\n     (6)在这个数据库里创建一张表时，表默认的字符集被设定为 character_set_database，也就是这个数据库默认的字符集；
\n     (7)当在表内设置一栏时，除非明确指定，否则此栏缺省的字符集就是表默认的字符集；
\n简单的总结一下，如果什么地方都不修改，那么所有的数据库的所有表的所有栏位的都用
\nlatin1 存储，不过我们如果安装 MySQL，一般都会选择多语言支持，也就是说，安装程序会自动在配置文件中把
\ndefault_character_set 设置为 UTF-8，这保证了缺省情况下，所有的数据库的所有表的所有栏位的都用 UTF-8 存储。
\n2.查看默认字符集(默认情况下，mysql的字符集是latin1(ISO_8859_1)
\n通常，查看系统的字符集和排序方式的设定可以通过下面的两条命令：
\n      mysql> SHOW VARIABLES LIKE 'character%';
\n+--------------------------+---------------------------------+
\n| Variable_name             | Value                            |
\n+--------------------------+---------------------------------+
\n| character_set_client      | latin1                           |
\n| character_set_connection | latin1                           |
\n| character_set_database    | latin1                           |
\n| character_set_filesystem | binary                      |
\n| character_set_results     | latin1                           |
\n| character_set_server      | latin1                           |
\n| character_set_system     | utf8                             |
\n| character_sets_dir        | D:"mysql-5.0.37"share"charsets" |
\n+--------------------------+---------------------------------+
\nmysql> SHOW VARIABLES LIKE 'collation_%';
\n+----------------------+-----------------+
\n| Variable_name         | Value            |
\n+----------------------+-----------------+
\n| collation_connection | utf8_general_ci |
\n| collation_database    | utf8_general_ci |
\n| collation_server      | utf8_general_ci |
\n+----------------------+-----------------+
\n3.修改默认字符集
\n(1) 最简单的修改方法，就是修改mysql的my.ini文件中的字符集键值，
\n如     default-character-set = utf8
\n       character_set_server = utf8
\n    修改完后，重启mysql的服务，service mysql restart
\n    使用 mysql> SHOW VARIABLES LIKE 'character%';查看，发现数据库编码均已改成utf8
\n+--------------------------+---------------------------------+
\n| Variable_name             | Value                            |
\n+--------------------------+---------------------------------+
\n| character_set_client      | utf8                             |
\n| character_set_connection | utf8                             |
\n| character_set_database    | utf8                             |
\n| character_set_filesystem | binary                           |
\n| character_set_results     | utf8                             |
\n| character_set_server      | utf8                             |
\n| character_set_system      | utf8                             |
\n| character_sets_dir        | D:"mysql-5.0.37"share"charsets" |
\n+--------------------------+---------------------------------+
\n    (2) 还有一种修改字符集的方法，就是使用mysql的命令
\n    mysql> SET character_set_client = utf8 ;
