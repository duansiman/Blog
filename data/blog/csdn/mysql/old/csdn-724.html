         软件开发中，一个好的工具的使用能让开发效率大幅度提升，由于SQL Server难安装性，易损坏性影响开发效率，从朋友那里借来了navicat，发现很实用的功能。

\n


\n

\n

Navicat是一个强大的MySQL数据库管理和开发工具。Navicat为专业开发者提供了一套强大的足够尖端的工具，但它对于新用户易于学 ，Navicat 已被誉为领导市场及最佳使用之MySQL 用户界面管理工具。

\n


\n

\n

它除了常见的建库建表，以及查询功能外，还支持表格打印，报表查询，报表打印，生成数据模型等功能，下面介绍两个常用功能：

\n

\n

            1. 导出所有表结构，使用sql语句查询出所有表结构，选择导出表类型，点击开始导出即可。

\n

   

SELECT\n\t表名 = CASE\nWHEN A.colorder = 1 THEN\n\tD.name\nELSE\n\t''\nEND,\n 表说明 = CASE\nWHEN A.colorder = 1 THEN\n\tisnull(F. VALUE, '')\nELSE\n\t''\nEND,\n 字段序号 = A.colorder,\n 字段名 = A.name,\n 字段说明 = isnull(G.[value], ''),\n 标识 = CASE\nWHEN COLUMNPROPERTY(A.id, A.name, 'IsIdentity') = 1 THEN\n\t'√'\nELSE\n\t''\nEND,\n 主键 = CASE\nWHEN EXISTS (\n\tSELECT\n\t\t1\n\tFROM\n\t\tsysobjects\n\tWHERE\n\t\txtype = 'PK'\n\tAND parent_obj = A.id\n\tAND name IN (\n\t\tSELECT\n\t\t\tname\n\t\tFROM\n\t\t\tsysindexes\n\t\tWHERE\n\t\t\tindid IN (\n\t\t\t\tSELECT\n\t\t\t\t\tindid\n\t\t\t\tFROM\n\t\t\t\t\tsysindexkeys\n\t\t\t\tWHERE\n\t\t\t\t\tid = A.id\n\t\t\t\tAND colid = A.colid\n\t\t\t)\n\t)\n) THEN\n\t'√'\nELSE\n\t''\nEND,\n 类型 = B.name,\n 占用字节数 = A.Length,\n 长度 = COLUMNPROPERTY(A.id, A.name, 'PRECISION'),\n 小数位数 = isnull(\n\tCOLUMNPROPERTY(A.id, A.name, 'Scale'),\n\t0\n),\n 允许空 = CASE\nWHEN A.isnullable = 1 THEN\n\t'√'\nELSE\n\t''\nEND,\n 默认值 = isnull(E. TEXT, '')\nFROM\n\tsyscolumns A\nLEFT JOIN systypes B ON A.xusertype = B.xusertype\nINNER JOIN sysobjects D ON A.id = D.id\nAND D.xtype = 'U'\nAND D.name <> 'dtproperties'\nLEFT JOIN syscomments E ON A.cdefault = E.id\nLEFT JOIN sys.extended_properties G ON A.id = G.major_id\nAND A.colid = G.minor_id\nLEFT JOIN sys.extended_properties F ON D.id = F.major_id\nAND F.minor_id = 0 --where d.name='OrderInfo' --如果只查询指定表,加上此条件\nORDER BY\n\tA.id,\n\tA.colorder

\n    \n

           2. 打印表模型

\n


\n

\n

         选择新建模型，将需要导出的表拖入右侧表格中，即可显示表格结构，右下角调整页面大小进行页面的缩放。
