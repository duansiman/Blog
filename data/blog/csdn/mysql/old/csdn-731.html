    接上篇介绍《基于CentOS的Mysql学习补充二--使用Shell创建数据库》，本文继续探索关于Shell和MySQL的结合使用，我不知道当一个数据库设计完成之后如何快速的创建设计好的数据库表和添加相应基本数据，我目前知道的就是使用Shell和SQL脚本来达到我的目的--快速的、多次的、可重复利用的创建数据库表。

\n

创建一个数据库表的SQL脚本：

\n

\n
/************************************************************\n#Author:\tchisj\n#Date:\t\t2015.7.22\n#Describe:\tCreate Database 'SmartCare' Table 'sct_Province'\n*************************************************************/\n\nUSE SmartCare;\n\nDROP TABLE IF EXISTS `sct_Province`;\n\nCREATE TABLE `sct_Province`(\n\t`ProvinceID` int(11) unsigned NOT NULL AUTO_INCREMENT,\n\t`ProvinceCode` varchar(11) DEFAULT NULL,\n\t`ParentID` varchar(11) DEFAULT NULL,\n\t`ProvinceName` varchar(50) DEFAULT NULL,\n\t`Level` tinyint(1) DEFAULT NULL,\n\tPRIMARY KEY (`ProvinceID`)\n) ENGINE=MyISAM AUTO_INCREMENT=1 DEFAULT CHARSET=utf8;

\n

\n
\n执行指定目录下所有的SQL脚本（指定目录，多个SQL脚本）的Shell脚本：\n
\n
\n
#!/bin/bash\n\n#Author: chisj\n#Time: 2015.7.22\n#Describe: Create Database Table\n\n#The username of mysql database\nUSER="root"\n\n#The password of mysql database\nPASS="dragonwake"\n\n#The datebase name will be created\nDATABASE="SmartCare"\n\nLOCATION=${PWD}\nCREATE_TABLE_SCT=create_table_sct_\n\nfor table_name in `ls ${LOCATION}/${CREATE_TABLE_SCT}*`\ndo\n\nmysql -u $USER -p$PASS << EOF > /dev/null\n\nSOURCE ${table_name};\n\nEOF\n\nif [ $? -eq 0 ]; then\n\t\n\techo "Create Table ${table_name} Success!"\n\t\nfi\n\ndone\n
其实关于Shell批量创建数据表的方法很多，之前又看到过，可惜没有记住0.0，可见记录很重要哟\n
\n
所以我还是按照我的思路重新弄了一遍：首先写好创建数据表的SQL脚本是必要的，可以将所有的都写在一个里面（个人觉得不好），也可以按照一个表写一个SQL脚本（目前我市采用这种方法），然后就是在指定目录下面按照指定SQL脚本名进行遍历，找到一个创建数据库表SQL脚本，然后就执行一次。
